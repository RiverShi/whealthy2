# Wealthy · 功能规格与后端 API

> 一句话定位：**以资产视角为核心、兼顾记账与净值追踪的个人财务管理系统。**
>
> 与普通记账 App 的差异：先有「资产负债台账」再有「收支流水」；支持多账本隔离与视图聚合；支持快照与净值变化追踪。

---

## 一、核心领域模型

### 1. 账本 `Book`

账本是资产与账目的数据集合，用户自由定义其含义（按资产类别、时间段、用途等）。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 主键 |
| name | string | 账本名称（用户自定义，App 不介入分类） |
| createdAt | datetime | |
| archivedAt | datetime? | 归档时间，归档后只读 |

---

### 2. 资产/负债条目 `Entry`

资产和负债的台账条目，是整个系统的核心实体。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | |
| bookId | string | 所属账本 |
| name | string | 条目名称 |
| kind | `asset \| liability` | 资产或负债 |
| isAccount | boolean | 是否作为记账账户（可关联凭证分录） |
| valuationType | `fixed \| manual` | 估值方式：固定值 / 手动估值 |
| value | number | 当前价值 |
| categoryL1Id | string? | 一级分类 |
| categoryL2Id | string? | 二级分类 |
| tagIds | string[] | 标签列表 |
| extra | JSON? | 扩展字段（卡券余额、赠送金、折旧参数等） |
| openedAt | datetime | 建立时间 |
| closedAt | datetime? | 关闭时间 |

> **关于 `isAccount`**：每个资产条目可选择性地作为记账账户（如现金、银行卡天然是账户；房产默认不是账户）。凭证分录可关联已标记为账户的条目，但当前阶段交易不自动驱动账户余额变更（仅预留关联关系）。

---

### 3. 条目价值调整记录 `EntryAdjustment`

记录资产条目价值的每次手动变更，形成价值变化历史。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | |
| entryId | string | 关联条目 |
| oldValue | number | 调整前价值 |
| newValue | number | 调整后价值 |
| reason | string? | 调整原因（估值修正、盘点差异等） |
| adjustedAt | datetime | |

---

### 4. 交易记录 `Transaction`

一笔交易的基本信息，包含时间和说明。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | |
| bookId | string | 所属账本 |
| description | string? | 备注说明 |
| happenedAt | datetime | 发生时间 |
| createdAt | datetime | |

---

### 5. 交易明细 `TransactionItem`

交易的具体明细行，采用简化的"从→到"模式，替代复杂的借贷记账。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | |
| transactionId | string | 所属交易 |
| type | `income \| expense \| transfer` | 收入 / 支出 / 转账 |
| amount | number | 金额 |
| categoryId | string? | 关联分类（收入/支出时必填） |
| fromAccountId | string? | 转出账户（支出/转账时必填） |
| toAccountId | string? | 转入账户（收入/转账时必填） |
| tagIds | string[] | 标签 |
| note | string? | 明细备注 |

**交易类型说明：**
- **收入 (income)**：需要指定收入类别 + 收入账户（资金流入哪个账户）
- **支出 (expense)**：需要指定支出类别 + 支出账户（资金从哪个账户流出）
- **转账 (transfer)**：账户间资金移动，指定转出账户 + 转入账户，不关联分类，不计入收支统计

**交易示例：**

```json
// 工资收入
Transaction: { description: "1月工资", happenedAt: "2025-01-25" }
  TransactionItem: { 
    type: "income", 
    amount: 8000, 
    categoryId: "工资收入", 
    toAccountId: "工资卡" 
  }

// 餐饮支出
Transaction: { description: "朋友聚餐", happenedAt: "2025-01-10" }
  TransactionItem: { 
    type: "expense", 
    amount: 200, 
    categoryId: "餐饮",
    fromAccountId: "现金"
  }

// 账户转账
Transaction: { description: "工资卡转零钱", happenedAt: "2025-01-15" }
  TransactionItem: { 
    type: "transfer", 
    amount: 1000,
    fromAccountId: "工资卡",
    toAccountId: "微信零钱"
  }
```

> **设计理念**：摒弃专业会计的借贷概念，采用更直观的"钱从哪来、到哪去"思维，降低记账门槛

---

### 6. 分类 `Category`

资产/负债/收支分类相互隔离，各自独立维护，采用两级体系。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | |
| domain | `asset \| liability \| income \| expense` | 所属领域（隔离管理） |
| level | `1 \| 2` | 一级或二级分类 |
| parentId | string? | 父分类 id（level=2 时必填） |
| name | string | |
| icon | string? | 图标 |

---

### 7. 标签 `Tag`

自由标签，按领域隔离（资产标签、负债标签、账目标签分开管理）。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | |
| domain | `asset \| liability \| transaction` | 所属领域 |
| name | string | |
| color | string? | 颜色 |

---

### 8. 统计视图 `View`

配置一组账本的聚合口径，实现跨账本汇总统计。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | |
| name | string | 视图名称 |
| bookIds | string[] | 纳入统计的账本列表 |

---

### 9. 快照 `Snapshot`

记录某时点的资产负债状态，支持全量和增量两种方式。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | |
| bookId | string | 所属账本 |
| type | `full \| incremental` | 全量 / 增量 |
| baseSnapshotId | string? | 增量基线快照 id |
| data | JSON | 快照数据（全量：完整条目值；增量：变化差集） |
| source | `auto \| manual` | 自动周期快照 / 用户手动创建 |
| createdAt | datetime | |

> **增量策略**：只记录与上一次快照相比发生变化的条目；每 N 次增量自动合并一次全量，避免回放链过长。

---

### 10. 快照任务 `SnapshotTask`

配置周期性自动快照，App 启动时懒检查并补齐缺失周期。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | |
| bookId | string | |
| frequency | `daily \| weekly \| monthly` | 快照频率 |
| lastRunAt | datetime? | 上次执行时间 |
| isActive | boolean | 是否启用 |

## 二、功能模块清单

| 模块 | 主要功能 |
|------|---------|
| **账本管理** | 创建 / 编辑 / 归档账本 |
| **资产/负债管理** | 条目 CRUD、手动调整价值、查看调整历史、按分类/标签筛选 |
| **记账** | 创建流水记录（Record），支持收入/支出/转账；可选将语义相关的多条记录组合为事件（Event） |
| **分类与标签** | 按领域独立管理两级分类树和自由标签 |
| **统计视图** | 配置跨账本视图、查看净资产、收支汇总、分类钻取明细 |
| **快照** | 手动创建快照、配置周期任务、查看历史净资产曲线、快照对比差值 |
| **App 启动检查** | 启动时执行懒快照检查，补齐缺失的周期快照 |

## 三、后端 API（Tauri Commands）

### 账本

```
list_books()
create_book(name)
update_book(id, name)
archive_book(id)
delete_book(id)
```

### 资产/负债条目

```
list_entries(book_id, filter?: { kind, is_closed, category_l1_id, tag_ids })
get_entry(id)
create_entry(book_id, name, kind, is_account, value, valuation_type,
             category_l1_id?, category_l2_id?, tag_ids?, extra?)
update_entry(id, ...)
delete_entry(id)

// 价值调整（自动写入 EntryAdjustment 记录）
adjust_entry_value(id, new_value, reason?)
list_entry_adjustments(entry_id)
```

### 流水记录

```
// 流水 CRUD
list_records(book_id, filter?: { record_type, event_id, category_id, from, to })
get_record(id)
create_record(book_id, type, amount, happened_at,
              event_id?, category_id?, from_account_id?, to_account_id?,
              tag_ids?, note?)
update_record(id, ...)    // patch 模式
delete_record(id)

// 事件 CRUD
list_events(book_id)
get_event(id)             // 含旗下全部流水
list_events(book_id)
create_event(book_id, name, description?)
update_event(id, name?, description?)
delete_event(id)          // 流水 event_id 自动置 null，不小流水
```

**Record 结构：**
```typescript
{
  type: 'income' | 'expense' | 'transfer',
  amount: number,           // 正数
  happenedAt: string,       // 每条流水自带时间
  eventId?: string,         // 可选，归属事件
  categoryId?: string,      // 收入/支出时填
  fromAccountId?: string,   // 支出/转账时填
  toAccountId?: string,     // 收入/转账时填
  tagIds?: string[],
  note?: string
}
```

### 分类

```
list_categories(domain, level?)
create_category(domain, level, name, parent_id?, icon?)
update_category(id, name?, icon?)
delete_category(id)
```

### 标签

```
list_tags(domain?)
create_tag(domain, name, color?)
update_tag(id, name?, color?)
delete_tag(id)
```

### 统计视图

```
list_views()
create_view(name, book_ids[])
update_view(id, name?, book_ids[]?)
delete_view(id)

// 统计查询
get_view_net_worth(view_id, at?)                            // 指定时点净资产
get_view_balance_sheet(view_id, at?)                        // 资产负债汇总
get_view_income_expense(view_id, from, to, group_by?)       // 收支汇总（按主/副分类）
get_view_entry_list(view_id, filter?)                       // 跨账本条目列表
```

### 快照

```
list_snapshots(book_id, from?, to?)
get_snapshot(id)
create_snapshot(book_id)                                    // 手动创建（全量）
diff_snapshots(from_id, to_id)                              // 两快照对比差值
```

### 快照任务

```
list_snapshot_tasks()
create_snapshot_task(book_id, frequency)
update_snapshot_task(id, frequency?, is_active?)
delete_snapshot_task(id)

// App 启动时调用，懒执行补齐缺失快照
check_and_run_snapshot_tasks()
```

## 四、关键设计决策

| 决策点 | 结论 |
|--------|------|
| 账本结构 | 极简，只有名称；币种、用途等由用户自行理解，App 不介入 |
| 账户属性 | 资产条目通过 `isAccount` 可选标记为账户，非强制绑定 |
| 记账模式 | **Record 优先**：每条流水独立存在、独立展示；Event 是可选的轻量聚合层，不绑定主从 |
| 分类隔离 | `asset / liability / income / expense` 四域各自维护独立分类树与标签池 |
| 统计口径 | 由 `View.bookIds` 决定；转账类型自动不计入收支统计 |
| 快照执行 | 周期快照无需后台常驻，App 启动时由 `check_and_run_snapshot_tasks` 懒补齐 |

## 附录：记账方案演进说明

**v1.0（初版）**：采用专业会计的凭证+分录模式，支持借贷平衡、主副分类、跨账本划转控制等复杂场景。

**v2.0（当前）**：重构为「流水为主体 + 事件为可选聚合」模式：
- **Record**（流水记录）= 原子单元，可独立展示，自带时间
- **Event**（事件）= 可选轻量聚合层，将语义相关的多条流水组合
- 事件不承载时间，删事件不删流水

**变更原因**：取消主从绑定关系，让每条记录可独立展示；同时支持将“购物+退款”、“聚餐AA”等关联场景轻量组合。
