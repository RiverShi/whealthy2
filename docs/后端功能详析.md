# 后端功能详析

## 重要说明
经过对代码的详细审查，发现当前实现与设计文档存在差异：`archive_book` 功能已在后端代码中移除，尽管数据库表中仍保留 `archived_at` 字段。

## 1. 后端已完成的功能

### 1.1 账本管理 (Books)

- 创建账本 (`create_book`)
- 列出账本 (`list_books`)
- 更新账本 (`update_book`)
- 删除账本 (`delete_book`)

注意：`archive_book` 功能已在当前代码中移除，虽然数据库结构中仍有 `archived_at` 字段但并未实现归档功能。

### 1.2 资产/负债条目管理 (Entries)

- 创建条目 (`create_entry`)
- 获取条目详情 (`get_entry`)
- 列出条目 (`list_entries`) - 支持过滤器
- 更新条目 (`update_entry`)
- 删除条目 (`delete_entry`)
- 调整条目价值 (`adjust_entry_value`) - 自动记录调整历史
- 列出价值调整历史 (`list_entry_adjustments`)

### 1.3 分类管理 (Categories)

- 列出分类 (`list_categories`) - 按领域和层级过滤
- 创建分类 (`create_category`) - 支持两级分类
- 更新分类 (`update_category`)
- 删除分类 (`delete_category`)

### 1.4 标签管理 (Tags)

- 列出标签 (`list_tags`) - 按领域过滤
- 创建标签 (`create_tag`)
- 更新标签 (`update_tag`)
- 删除标签 (`delete_tag`)

### 1.5 事件管理 (Events)

- 列出事件 (`list_events`)
- 获取事件详情 (`get_event`) - 包含相关流水记录
- 创建事件 (`create_event`)
- 更新事件 (`update_event`)
- 删除事件 (`delete_event`) - 流水记录的 event_id 会自动置为 NULL

### 1.6 流水记录管理 (Records)

- 列出记录 (`list_records`) - 支持多种过滤条件
- 获取记录详情 (`get_record`)
- 创建记录 (`create_record`) - 支持收入/支出/转账类型
- 更新记录 (`update_record`)
- 删除记录 (`delete_record`)
- 获取时间轴视图 (`list_feed`) - 按时间排序的流水列表

### 1.7 统计功能 (Statistics)

- 获取账本统计数据 (`get_book_stats`) - 资产/负债/收支统计

### 1.8 快照功能 (Snapshots)

- 列出快照 (`list_snapshots`) - 支持时间范围过滤
- 获取快照详情 (`get_snapshot`)
- 创建快照 (`create_snapshot`) - 手动创建全量快照
- 比较快照 (`diff_snapshots`) - 两个快照之间的差异
- 创建快照任务 (`create_snapshot_task`) - 定期快照
- 列出快照任务 (`list_snapshot_tasks`)
- 获取账本的快照任务 (`get_snapshot_task_for_book`)
- 更新快照任务 (`update_snapshot_task`)
- 删除快照任务 (`delete_snapshot_task`)
- 检查并运行快照任务 (`check_and_run_snapshot_tasks`) - 启动时懒执行

## 3. 数据库详细结构 (src-tauri/src/db.rs)

### 3.1 books 表 - 账本表

```sql
CREATE TABLE IF NOT EXISTS books (
    id          TEXT PRIMARY KEY,      -- 唯一标识符
    name        TEXT NOT NULL,         -- 账本名称
    created_at  TEXT NOT NULL          -- 创建时间
);
```

### 3.2 categories 表 - 分类表

```sql
CREATE TABLE IF NOT EXISTS categories (
    id        TEXT PRIMARY KEY,        -- 唯一标识符
    domain    TEXT NOT NULL CHECK(domain IN ('asset','liability','income','expense')), -- 领域：资产/负债/收入/支出
    level     INTEGER NOT NULL CHECK(level IN (1,2)), -- 层级：1级或2级
    parent_id TEXT REFERENCES categories(id) ON DELETE CASCADE, -- 父分类ID（2级分类时必填）
    name      TEXT NOT NULL,           -- 分类名称
    icon      TEXT                     -- 图标
);
```

### 3.3 tags 表 - 标签表

```sql
CREATE TABLE IF NOT EXISTS tags (
    id     TEXT PRIMARY KEY,           -- 唯一标识符
    domain TEXT NOT NULL CHECK(domain IN ('asset','liability','transaction')), -- 领域：资产/负债/账目
    name   TEXT NOT NULL,              -- 标签名称
    color  TEXT                        -- 颜色
);
```

### 3.4 entries 表 - 条目表（资产/负债）

```sql
CREATE TABLE IF NOT EXISTS entries (
    id              TEXT PRIMARY KEY,  -- 唯一标识符
    book_id         TEXT NOT NULL REFERENCES books(id) ON DELETE CASCADE, -- 所属账本
    name            TEXT NOT NULL,     -- 条目名称
    kind            TEXT NOT NULL CHECK(kind IN ('asset','liability')), -- 类型：资产或负债
    is_account      INTEGER NOT NULL DEFAULT 0, -- 是否作为记账账户
    valuation_type  TEXT NOT NULL CHECK(valuation_type IN ('fixed','manual')), -- 估值方式：固定/手动
    value           REAL NOT NULL DEFAULT 0, -- 当前价值
    category_l1_id  TEXT REFERENCES categories(id), -- 一级分类
    category_l2_id  TEXT REFERENCES categories(id), -- 二级分类
    extra           TEXT,              -- 扩展字段（JSON格式）
    opened_at       TEXT NOT NULL,     -- 建立时间
    closed_at       TEXT               -- 关闭时间
);
```

### 3.5 entry_tags 表 - 条目标签关联表

```sql
CREATE TABLE IF NOT EXISTS entry_tags (
    entry_id TEXT NOT NULL REFERENCES entries(id) ON DELETE CASCADE, -- 条目ID
    tag_id   TEXT NOT NULL REFERENCES tags(id) ON DELETE CASCADE,   -- 标签ID
    PRIMARY KEY (entry_id, tag_id)     -- 复合主键
);
```

### 3.6 entry_adjustments 表 - 条目价值调整记录表

```sql
CREATE TABLE IF NOT EXISTS entry_adjustments (
    id          TEXT PRIMARY KEY,      -- 唯一标识符
    entry_id    TEXT NOT NULL REFERENCES entries(id) ON DELETE CASCADE, -- 关联条目
    old_value   REAL NOT NULL,         -- 调整前价值
    new_value   REAL NOT NULL,         -- 调整后价值
    reason      TEXT,                  -- 调整原因
    adjusted_at TEXT NOT NULL          -- 调整时间
);
```

### 3.7 events 表 - 事件表

```sql
CREATE TABLE IF NOT EXISTS events (
    id          TEXT PRIMARY KEY,      -- 唯一标识符
    book_id     TEXT NOT NULL REFERENCES books(id) ON DELETE CASCADE, -- 所属账本
    name        TEXT NOT NULL,         -- 事件名称
    description TEXT,                  -- 描述
    created_at  TEXT NOT NULL          -- 创建时间
);
```

### 3.8 records 表 - 流水记录表

```sql
CREATE TABLE IF NOT EXISTS records (
    id              TEXT PRIMARY KEY,  -- 唯一标识符
    book_id         TEXT NOT NULL REFERENCES books(id) ON DELETE CASCADE, -- 所属账本
    event_id        TEXT REFERENCES events(id) ON DELETE SET NULL, -- 关联事件（可空）
    type            TEXT NOT NULL CHECK(type IN ('income','expense','transfer')), -- 类型：收入/支出/转账
    amount          REAL NOT NULL CHECK(amount > 0), -- 金额（必须大于0）
    category_id     TEXT REFERENCES categories(id), -- 关联分类（收入/支出时必填）
    from_account_id TEXT REFERENCES entries(id), -- 转出账户（支出/转账时必填）
    to_account_id   TEXT REFERENCES entries(id), -- 转入账户（收入/转账时必填）
    note            TEXT,              -- 备注
    happened_at     TEXT NOT NULL,     -- 发生时间
    created_at      TEXT NOT NULL      -- 创建时间
);
```

### 3.9 record_tags 表 - 流水记录标签关联表

```sql
CREATE TABLE IF NOT EXISTS record_tags (
    record_id TEXT NOT NULL REFERENCES records(id) ON DELETE CASCADE, -- 记录ID
    tag_id    TEXT NOT NULL REFERENCES tags(id) ON DELETE CASCADE,   -- 标签ID
    PRIMARY KEY (record_id, tag_id)    -- 复合主键
);
```

### 3.10 views 表 - 统计视图表

```sql
CREATE TABLE IF NOT EXISTS views (
    id   TEXT PRIMARY KEY,              -- 唯一标识符
    name TEXT NOT NULL                  -- 视图名称
);
```

### 3.11 view_books 表 - 视图账本关联表

```sql
CREATE TABLE IF NOT EXISTS view_books (
    view_id TEXT NOT NULL REFERENCES views(id) ON DELETE CASCADE, -- 视图ID
    book_id TEXT NOT NULL REFERENCES books(id) ON DELETE CASCADE, -- 账本ID
    PRIMARY KEY (view_id, book_id)      -- 复合主键
);
```

### 3.12 snapshots 表 - 快照表

```sql
CREATE TABLE IF NOT EXISTS snapshots (
    id               TEXT PRIMARY KEY, -- 唯一标识符
    book_id          TEXT NOT NULL REFERENCES books(id) ON DELETE CASCADE, -- 所属账本
    type             TEXT NOT NULL CHECK(type IN ('full','incremental')), -- 类型：全量/增量
    base_snapshot_id TEXT REFERENCES snapshots(id), -- 基线快照ID（增量快照时使用）
    data             TEXT NOT NULL,    -- 快照数据（JSON格式）
    source           TEXT NOT NULL CHECK(source IN ('auto','manual')), -- 来源：自动/手动
    created_at       TEXT NOT NULL     -- 创建时间
);
```

### 3.13 snapshot_tasks 表 - 快照任务表

```sql
CREATE TABLE IF NOT EXISTS snapshot_tasks (
    id           TEXT PRIMARY KEY,      -- 唯一标识符
    book_id      TEXT NOT NULL REFERENCES books(id) ON DELETE CASCADE, -- 所属账本
    frequency    TEXT NOT NULL CHECK(frequency IN ('daily','weekly','monthly')), -- 频率：每日/每周/每月
    last_run_at  TEXT,                  -- 上次运行时间
    is_active    INTEGER NOT NULL DEFAULT 1 -- 是否激活（1=激活，0=禁用）
);
```

## 4. 数据库约束和关系

- **外键约束**：确保数据完整性，删除父记录时会自动删除相关子记录（CASCADE）
- **CHECK约束**：限制字段值的有效范围
- **PRIMARY KEY**：每个表都有唯一主键
- **复合主键**：用于多对多关系表
- **PRAGMA设置**：WAL模式（提高并发性能）和外键约束启用

## 5. 数据库索引
